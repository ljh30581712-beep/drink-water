## 1. 문제 정의

반도체 팹(clean room) 생산라인 근로자는 방진복을 입은 채 2~3교대로 몇 시간씩 서서 정밀 작업을 이어갑니다. 방진복을 벗고 다시 입는 과정이 번거로워 물 마시는 순간을 계속 미루게 되고, 반복 작업에 집중하다 보면 시간 감각이 무뎌져 물을 안 마신 지 몇 시간이 지났는지조차 인지하지 못하는 경우가 많습니다.

이렇게 쌓인 수분 부족은 집중력·판단력 저하로 이어져 정밀 작업 중 실수를 유발하고, 장시간 근무와 겹치면 어지럼증이나 안전사고 위험까지 커집니다. 기존의 단순 알람 앱은 "알람을 껐다"와 "실제로 물을 마셨다" 사이의 간극을 확인할 방법이 없다는 근본적인 한계가 있습니다.

**HydroCheck는 이 간극을 AI Vision으로 메웁니다.** 정해진 시간에 알람이 울리고, 사용자가 실제로 물을 마시는 모습을 촬영해야만 AI가 이를 판별해 알람을 해제합니다. 단순히 "알림"이 아니라 "행동 검증"을 통해 습관을 강제하는 것이 핵심 차별점입니다.

## 2. 아키텍처

브라우저(프론트엔드)는 AI API 키를 전혀 알지 못합니다. 사진을 자체 백엔드 서버로만 전송하고, 백엔드가 환경변수에 저장된 키로 AI API를 대신 호출합니다. 이렇게 설계한 이유는 API 키가 클라이언트 코드에 노출되면 누구나 탈취해 악용할 수 있기 때문입니다.

```
[사용자 브라우저]                [Vercel 서버리스 함수]              [Google Gemini API]
   index.html      --사진(JPEG)-->   api/verify.js      --이미지+프롬프트-->   gemini-3.5-flash
   (알람 타이머,                    (API 키는 여기에만                        (음수 여부 판별,
    카메라 촬영,                     환경변수로 보관,                          이유 텍스트 반환)
    다국어 UI)      <--판별 결과---   브라우저에 노출 X)   <---JSON 결과-----
```

- 촬영된 이미지는 판별 즉시 폐기되며, 서버나 데이터베이스 어디에도 저장되지 않습니다.
- 알람음/진동은 브라우저 자동재생 정책을 고려해, 사용자가 타이머를 시작하는 클릭 시점에 오디오 컨텍스트를 미리 활성화해두는 방식으로 구현했습니다.

## 4. 실행 방법

### 바로 체험하기 (권장)
라이브 URL로 접속 → "타이머 시작" 클릭 → "지금 알람 테스트(심사용)" 버튼으로 즉시 알람 체험 가능 → 카메라로 물 마시는 모습 촬영 → AI 판별 결과 확인

### 로컬에서 실행하기
```bash
# 1. 저장소 클론
git clone <이 저장소 URL>
cd hydrocheck

# 2. Google AI Studio(https://aistudio.google.com/apikey)에서 Gemini API 키 발급

# 3. Vercel CLI로 로컬 실행 (환경변수 필요)
npm i -g vercel
vercel dev
# 실행 시 GEMINI_API_KEY 환경변수를 등록해야 합니다
```

### 배포하기
1. GitHub 저장소를 Vercel에 Import
2. Vercel 프로젝트 Settings → Environment Variables에 `GEMINI_API_KEY` 등록
3. Deploy

## 5. AI 활용 명세

| 항목 | 내용 |
|---|---|
| 사용 AI 모델 | Google Gemini (`gemini-3.5-flash`) |
| 활용 방식 | 사용자가 촬영한 사진 1장을 Vision(이미지 이해) 기능으로 분석해 "물/음료를 마시는 동작인지" 판별. 판별 결과(true/false)와 짧은 판별 이유를 JSON으로 반환받아 알람 해제 여부를 결정 |
| 활용 범위 | 핵심 기능(음수 인증) 전체가 AI 판별에 의존 — AI 없이는 서비스 성립 불가 |
| 개발 보조 | 코드 초안 작성 및 디버깅(구조 설계, 오류 진단) 과정에서 Claude(Anthropic)를 개발 보조 도구로 활용. 최종 코드는 팀원이 검토·수정함 |
| 오픈소스/외부 서비스 | Vercel(배포), GitHub(코드 저장소) |

## 6. 개인정보 및 윤리

- 촬영된 이미지는 **AI 판별 목적으로 1회 전송 후 즉시 폐기**되며, 서버나 별도 저장소에 보관되지 않습니다.
- 얼굴 인식이나 신원 식별 기능은 없으며, "음수 행동 여부"만 판별합니다.
- 본 서비스는 성인 근로자(반도체 팹 교대근무자)를 대상으로 설계되었으며, 미성년자를 대상으로 한 서비스가 아닙니다.
- AI 생성 콘텐츠 표기: 위 "5. AI 활용 명세" 참조.

## 7. 다국어 지원

한국어 / English / 日本語 3개 언어를 화면 내 버튼으로 즉시 전환할 수 있습니다. UI 텍스트뿐 아니라 AI 판별 결과 문구도 선택된 언어로 반환되도록 서버와 연동했습니다.

## 8. 개발 히스토리 (이슈 대응 기록)

개발 과정에서 겪은 주요 문제와 해결 과정입니다.

| 단계 | 문제 | 해결 |
|---|---|---|
| 1차 → 2차 구조 전환 | 프론트엔드에서 AI API를 직접 호출하는 구조라 배포 환경 밖(다른 브라우저/뷰어)에서 작동 불가 | 프론트엔드-백엔드 분리 아키텍처로 전면 개편, API 키를 서버에만 보관 |
| AI 판별 실패 | AI가 한국어로 응답 시 영어 키워드("YES")만 인식하는 파싱 로직 때문에 오탐 발생 | JSON 구조화 응답으로 전환, 판정 기준을 관대하게 조정 |
| 배포 오류 | GitHub 모바일 업로드 시 `api/` 폴더 구조가 깨져 서버리스 함수 인식 실패 | 파일 경로를 직접 지정해 폴더 구조 재생성 |
| 504 타임아웃 | Vercel 기본 함수 실행 제한(10초)을 AI 분석 시간이 초과 | `vercel.json`의 `maxDuration` 값 상향 조정 |
| 알람 무음 문제 | 브라우저 자동재생 정책으로 타이머 자동 알람 시 소리가 재생되지 않음 | 사용자의 타이머 시작 클릭(정상적인 사용자 제스처) 시점에 오디오 컨텍스트를 미리 활성화 |
